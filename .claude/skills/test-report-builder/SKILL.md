# Test Report Builder 스킬

`results.json`과 캡처된 아티팩트를 **Playwright 스타일 HTML 리포트** + **마크다운 요약**으로 변환한다.

참고: Playwright MCP는 테스트 러너를 거치지 않으므로 러너 기본 HTML 리포트가 자동 생성되지 않는다. 그래서 번들 생성기(`build-report.mjs`)로 동일한 정보(케이스/단계/통계/증거)를 담은 Playwright 스타일 정적 HTML 리포트를 생성한다.

## 워크플로우

### 1. HTML 리포트 생성 (결정적 — 스크립트)
```bash
node .claude/skills/test-report-builder/scripts/build-report.mjs _workspace/02_results/results.json --out report
```
- `report/index.html` 생성. 의존성 없음(순수 Node).
- 실패 케이스를 상단에, 케이스별로 단계 결과·에러·스크린샷/비디오/트레이스 링크를 포함한다.
- 스크린샷은 `<img>`로 임베드, 비디오/트레이스는 링크로 연결(상대경로).

### 2. 마크다운 요약 작성 (서술 — 직접)
`report/summary.md`를 직접 작성한다. 포함 항목:
- **핵심 지표**: 총 N / 통과 N (통과율 %) / 실패 N / 스킵 N, 실행 시각·대상 URL
- **실패 케이스 분석**: 각 실패에 대해 — 어느 단계에서, 무엇이 기대와 달랐는지, 추정 원인(셀렉터 변경/타이밍/기능 결함/데이터)
- **스킵 사유**: 왜 수행 불가였는지 (예: 로그인 실패로 의존 케이스 스킵)
- **권장 조치**: 우선순위순으로 (재현 방법, 확인할 화면, 의심 영역)
- **HTML 리포트 경로** 안내

작성 원칙:
- 실패를 전면에. 통과 나열보다 "무엇이 깨졌고 왜인가"에 집중한다.
- 각 항목을 원본 테스트 케이스 ID로 역참조한다.
- 추정에는 근거(스크린샷/에러 메시지)를 붙인다.

### 3. 자격증명 비노출 점검
`results.json`에 민감값이 평문으로 남아 있으면 리포트 생성 전에 마스킹한다(`****`). 비밀번호·토큰·쿠키 값이 단계 note나 error에 섞이지 않았는지 확인한다.

## 출력
- `report/index.html` — 시각적 HTML 리포트
- `report/summary.md` — 사람이 읽는 요약·분석
- 최종 보고: 리포트 경로 + 통과율 + 실패 건수

## 에러 핸들링
- `results.json` 누락/손상: executor 산출물 확인 요청. 부분 결과로 생성하고 누락 명시.
- 아티팩트 경로 깨짐: 리포트에 "증거 누락" 표기하되 생성은 진행.
